Overview

The Accessibility Bookmarklets are visualization tools that provide web accessibility testers with an initial view that indicates whether accessibility issues exist on a web page. For example, the Headings bookmarklet makes it readily apparent when heading levels are inappropriately skipped.

Product Definition

The stakeholders, in conjunction with the design team, determined early on that the bookmarklets were not intended to provide detailed web accessibility evaluation data. Rather, they would allow anyone with an interest in web accessibility to become familiar with certains categories of accessibility requirements.

These categories would include heading structure, landmark regions, form control labels, list markup for grouping sequences of related items, and image alternative text.

They are easy to install and provide an approachable lightweight visual assessment. Because they are written in pure JavaScript, they are compatible with all modern browsers and are typically installed as bookmarks in the browser's 'Favorites' toolbar.
